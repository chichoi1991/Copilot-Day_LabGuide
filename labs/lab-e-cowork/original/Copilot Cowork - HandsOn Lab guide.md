## 1단계: 준비 — M365 Copilot 라이선스 및 Copilot Cowork 확인

### 1-1. 라이선스 확인

Copilot Cowork 기능을 사용하려면 **Microsoft 365 Copilot 라이선스 및 Consumptive Plan ** 적용이 필요합니다.

> \*\*Tip:\*\* Copilot Chat 페이지(`microsoft365.com/copilot`) 왼쪽 상단 Work IQ가 보인다면 라이선스가 활성화된 상태입니다.

### 1-2. Copilot Cowork 확인

1. 왼쪽 메뉴에서 Cowork가 접근된다면 Cowork가 활성화된 상태입니다.
2. 오른쪽 상단에서 LLM 모델 변경이 가능합니다.
> 기본으로 Auto, 일상 작업에는 Sonnet, 복잡한 작업에는 Opus! 앞으로는 GPT 5.5 모델도 들어올 예정입니다.

### 내 OneDrive의 개인 스킬 위치

개인 스킬 파일은 OneDrive의 다음 경로에 저장됩니다:

```Text
Documents/Cowork/
```

> 파일 관리자(OneDrive 웹 또는 파일 탐색기)로 이 폴더에 접근해 기존 스킬을 확인하고 추가할 수 있습니다.

## 3단계: 나만의 SKILL 작성

### SKILL.md 기본 구조

```Text
---
name: skill-name-
description: 이 스킬이 언제 실행되어야 하는지 한 줄 설명 (Copilot이 트리거 판단에 사용)
triggers:
  - "트리거 문장 예시 1"
  - "트리거 문장 예시 2"
---

## 목적
이 스킬이 하는 일을 간단히 설명합니다.

## 실행 지침
Copilot이 이 스킬을 실행할 때 따라야 할 구체적인 지침을 작성합니다.

1. 첫 번째 단계
2. 두 번째 단계
3. 결과물 형식

## 출력 형식
원하는 결과물의 형식이나 템플릿을 정의합니다.
```

## 방법 1: 채팅으로 SKILL 등록 

채팅창에서 아래 프롬프트로 직접 스킬을 만들고 저장할 수 있습니다.

### Step A — 스킬 생성 요청

```
보고서 PPT 템플릿을 만드는 새 스킬을 만들어줘.

이름: exec-report-ppt
설명:  보고용 EV 자동차 품질팀 PPT 템플릿
트리거: "대표 보고 PPT 만들어줘", "임원 보고 슬라이드 만들어줘", "대표 보고서 PPT 생성해줘", "경영 보고 PPT 만들어줘", "보고 자료 PPT로 만들어줘"

지침:
이 스킬이 실행되면 아래 사양에 따라 한국 대기업 보고서 스타일의 PowerPoint 파일을 생성한다. 사용자가 첨부한 파일을 참조하여 해당 문서에 내용이 자세히 적히도록 슬라이드를 채운다. 데이터가 없는 항목은 삭제하지 말고 편집 가능한 플레이스홀더(\[내용 입력 필요])로 남긴다. 

1. 기본 조건

   1. 슬라이드 크기: A4 Landscape, 11.69 × 8.27 inch.
   * 전체 배경: 흰색. 사진 배경 또는 장식 이미지는 사용하지 않는다.
   2. 모든 텍스트, 도형, 표, 차트는 PPT에서 직접 편집 가능한 객체로 만든다. 배경 이미지로 고정하지 않는다.
   3. 폰트: Malgun Gothic 혹은 맑은 고딕
   4. 메인 색상: Deep Blue #1844A3, Accent Green #009653, Issue Background #EFF7FF, Pale Blue #F6FAFF, Body Gray #3F4147, Mid Gray #7B8190, Line Gray #DDE5EE.

|역할|이름|HEX|
|주조색|Deep Blue|#1844A3|
|강조색|Accent Green|#009653|
|슬라이드 배경|Background|#FFFFFF|
|연한 배경|Pale Blue|#F6FAFF|
|본문 텍스트|Body Gray|#3F4147|
|보조 텍스트|Mid Gray|#7B8190|
|구분선·테두리|Line Gray|#DDE5EE|

2. 공통 헤더/푸터 규칙 (전 슬라이드 동일 적용)

  상단 4단 탭 네비게이션

* 위치: 상단 여백 y=0.22in
* 탭 텍스트: "I. 사업 배경" / "II. 추진 전략" / "III. 수행 방법" / "IV. 제안 사항"
* 탭 시작: x=0.76in, 탭 폭: 1.56in, 탭 간격: 0.13in, 탭 높이: 0.22in

  우측 상단 문서 정보

* 위치: x=9.1in 근처, 우측 정렬
* 내용: 문서명 + 페이지 번호
* 서식: 5.7pt, 회색

  헤더 하단 구분선

* 위치: y=0.62in, x=0.72in \~ 10.97in
* 색상: Accent Green(#009653), 선 굵기 1.3pt

  하단 푸터

* 위치: y=7.85in
* 왼쪽: Confidential – For Proposal Only — 4.9pt, Mid Gray
* 오른쪽: Copyright 문구 — 4.9pt, Mid Gray
* 구분선: 얇은 그린-그레이 수평선

3. 제목 영역 규칙 (공통)

* 본문 시작: x=0.78in
* 섹션 라벨: y=0.83in, 7pt, Body Gray(#3F4147)
* 대제목: y=1.03in, 20–22pt, Bold, Deep Blue(#1844A3)
* 설명문: y=1.46in

  * 좌측: Accent Green 세로 바 (폭 0.05in × 높이 0.18in)
  * 설명문: 10pt, Body Gray(#3F4147)

4. Why? 박스 규칙

  * 위치·크기: x=0.78in, y=1.82in, w=10.13in, h=0.38in
  * 배경: #EFF7FF
  * 좌측 라벨: "Why?", Deep Blue Bold 8pt, x=0.91in
  * 메시지 2개:

    * 첫 번째: x=1.78in
    * 두 번째: x=6.32in
  * 각 메시지는 ·로 시작, 8.4pt Bold, 색상 #34405A

5. 핵심 컴포넌트 규격

  * 2×2 이슈 그리드: 중앙 세로선 x=5.94in, y=2.42\~6.93in. 좌측 모듈 x=0.84in w=4.78in, 우측 모듈 x=6.30in w=4.63in. 상단 y=2.54in, 하단 y=5.02in, 모듈 높이 1.55in.
  * 모듈: 번호 박스 0.37×0.29in, 번호 01/03은 Deep Blue, 02/04는 Green. 제목은 12.8pt Bold Deep Blue. 제목 아래 얇은 #DDE5EE 라인. 본문 bullet은 8.7pt Body Gray.
  * KPI Card: 흰 배경, #DDE5EE 라인, 값은 15pt Bold, 라벨은 7pt Mid Gray.
  * 표: 헤더 배경 #EFF7FF, 헤더 텍스트 Deep Blue Bold, 본문은 흰색/#F8FAFC 교차 배경.

6. 생성해야 할 슬라이드 구성

  * 최대 10장으로 만든다. 제목, 목차, 결론 슬라이드를 반드시 포함한다.
  * 1)표지, 2) 목차, 3) 섹션 구분, 4) 사업 추진 배경 2×2 이슈 그리드, 5) Executive Summary, 6) 현황 진단 대시보드 및 Gap 분석, 7) 추진 전략 방향, 8) 대안 비교 및 선정, 9) 기대효과, 10) 끝맺음.

7. 문체와 콘텐츠 규칙

  * 한국 대기업 보고서 톤으로 작성한다.
  * 슬라이드당 핵심 메시지는 1개, 보조 메시지는 2\~4개로 제한한다.
  * 텍스트는 실제 보고서 예시 문장으로 채우되, 사용자가 쉽게 바꿀 수 있도록 과도하게 길게 쓰지 않는다.
  * 제목은 Deep Blue Bold, 강조 수치나 실행 포인트는 Green을 사용한다.

8. 품질 검수
   * 카드·차트 등 객체가 PPT에서 개별 편집 가능한지 확인한다.
   * 플레이스홀더가 누락 없이 모든 슬라이드에 배치되었는지 점검한다.
   * 진행 표시 바가 각 섹션 슬라이드에서 올바른 위치를 활성화하는지 확인한다.
   * 폰트, 컬러 팔레트가 전 슬라이드에 일관 적용되었는지 검수한다.
```

### Step B — 스킬 저장 확인

```

```

> \*\*OneDrive 경로:\*\* `Documents/Coworker/<skill-name>/SKILL.md` 에 자동 저장됩니다.

스킬 품질 점수 채점 확인하고 점수를 더 올리고싶으면

```

```

## 방법 2: OneDrive에 직접 SKILL.md 파일 업로드

Copilot 채팅 없이 파일을 직접 작성해 업로드하는 방법입니다.

### Step A — .md 파일 OneDrive에 업로드

**방법 A (웹 브라우저):**

1. 새 탭에서 [Sharepoint.com](https://absx67373023.sharepoint.com/Shared%20Documents/Forms/AllItems.aspx?id=%2FShared%20Documents%2FCopilot%5Fand%5FAgent%20Day&viewid=e257e0cf%2D1b2f%2D40f8%2Db8e2%2D7e2d086a0a61) 를 접속하고 Lab E > Skill.md 파일 다운로드
2. [onedrive.cloud.com](https://onedrive.cloud.microsoft) 접속
3. `My files > Documents > Coworker > skills` 폴더로 이동
4. `새로 만들기 > 폴더` 로 스킬 이름 폴더 생성 
	폴더 이름:
	```
	ev-trend-dashboard-html
	```
5. 생성한 폴더 안에 `SKILL.md` 파일 업로드

**방법 B (파일 탐색기):**

1. 동기화된 OneDrive 폴더 열기
2. `Documents > Coworker > skills` 경로로 이동
3. 스킬 폴더 생성 후 `SKILL.md` 파일 이동

> \*\*주의:\*\* 폴더명은 `name:` 필드의 값과 일치시키는 것을 권장합니다.

스킬을 등록한 후 아래 프롬프트로 검증합니다.

### 4-1. 스킬 목록에서 확인

```
내 스킬 목록을 보여줘
```

새로 등록한 스킬이 목록에 나타나는지 확인합니다.

### 4-2. 스킬 상세 검증

```
 스킬을 검증해줘
```

또는:

```
내 스킬 파일에 문제가 있는지 확인해줘
```

Copilot이 확인하는 항목:

|검사 항목|정상 상태|
|-|-|
|YAML 프론트매터 형식|`---` 블록이 올바르게 닫혀 있음|
|`name` 필드|영문 소문자, 하이픈만 사용|
|`description` 필드|한 줄, 명확한 설명|
|`triggers` 목록|2개 이상의 트리거 문장|
|스킬 본문|실행 지침이 충분히 구체적임|
|트리거 충돌|다른 스킬과 트리거가 겹치지 않음|

## 5단계: SKILL 사용하기

### 5-1. 트리거 문장으로 호출

> 새 채팅으로 시작합니다.

아래 프롬프트를 채팅 창에 입력합니다.:

```
EV 자동차 시장에 대한 최근 한달 내의 뉴스 주요 포인트를 정리해서 EV 트렌드 대시보드 만들어서 나에게 메일로 전송해. 
메일 내용은 내가 검수하고 본부장님에게 보낼 수 있도록 보고 스타일로 작성해줘. 
```

```
그리고 EV 자동차 관련 한달 내의 최신 뉴스와 함께 우리 내부 EV 충전소 확장을 위해서 /EV 충전소 데이터_2025.xlsx 와 EV 충전소 인사이트.docx
문서를 참고하여 PPT 보고서를 만들어줘. 
```

### 5-2. 슬래시 명령어로 직접 호출

스킬 이름 앞에 `/`를 붙여 직접 실행합니다:

```
/exec-report-ppt 을 사용하여 최신 전기 자동차 산업 조사하고 보고서 만들어줘.
```

### 5-3. 스킬 수정

스킬 동작을 개선하고 싶을 때:

```
exec-report-ppt 스킬을 수정해줘.
맨 마지막 슬라이드에 end of document를 추가해줘
```

### 5-4. 스킬 삭제

```
exec-report-ppt 스킬을 삭제해줘
```

## 자주 묻는 질문 (FAQ)

**Q. 스킬이 목록에 나타나지 않아요.**  
A. OneDrive 동기화가 완료될 때까지 1-2분 기다린 후 다시 확인하세요. 폴더 구조(`Documents/Cowork/\[스킬명]/SKILL.md`)가 정확한지도 점검하세요.

**Q. 트리거 문장을 입력해도 스킬이 실행되지 않아요.**  
A. 트리거가 너무 짧거나 일반적인 표현이면 시스템 기본 동작이 우선 실행될 수 있습니다. 트리거를 더 구체적이고 독특한 표현으로 변경해 보세요.

**Q. 스킬 본문이 너무 길면 안 되나요?**  
A. 스킬 지침은 **명확하고 구체적**일수록 좋습니다. 단, 지나치게 복잡하면 Copilot이 지침 전체를 따르지 못할 수 있으니 핵심 단계를 5-7개 이내로 정리하는 것을 권장합니다.

**Q. 여러 스킬을 동시에 사용할 수 있나요?**  
A. 한 번에 하나의 스킬이 실행됩니다. 복합 작업이 필요하면 하나의 스킬 안에 여러 단계를 통합하거나, 대화를 이어가며 순차적으로 실행하세요.
