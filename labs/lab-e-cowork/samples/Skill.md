
이름: ev-trend-dashboard-html

설명: 인터넷에서 수집한 신규 EV 자동차 현황·트렌드·시사점을 한눈에 보는 다크 모드 HTML 대시보드 템플릿

트리거: "EV 트렌드 대시보드 만들어줘", "신규 전기차 현황 HTML로 만들어줘", "EV 시장 동향 페이지 만들어줘", "전기차 트렌드 대시보드 생성해줘", "새 EV 자동차 정리 대시보드 만들어줘"

지침:
이 스킬이 실행되면 인터넷에서 수집한 최신 EV 자동차 정보를 정리한 단일 HTML 대시보드를 생성한다. 새로 출시되었거나 공개된 EV 모델들의 현황을 모으고, 핵심 트렌드와 시사점을 한눈에 파악할 수 있는 다크 모드 기반 반응형 레이아웃을 지향한다. 최신 데이터는 웹 검색 결과를 참조해 채우고, 출처가 불명확하거나 비어 있는 항목은 삭제하지 말고 편집 가능한 플레이스홀더([출처/데이터 입력])로 남긴다.

1. 기본 조건
   * 단일 .html 파일로 생성하며, CSS는 <style> 안에 인라인으로 포함한다
   * 차트가 필요한 경우 Chart.js를 CDN으로 불러온다
   * 반응형 레이아웃(CSS Grid / Flexbox)으로 데스크탑·태블릿 모두 대응한다
   * 폰트: Pretendard 또는 system-ui
   * 컬러 변수(CSS :root):

   |역할|변수명|HEX|
   |---|---|---|
   |배경|--bg|#10172A|
   |카드 배경|--card|#1C2640|
   |주조색(Cyan)|--cyan|#22D3EE|
   |강조색(Lime)|--lime|#A3E635|
   |경고색(Amber)|--amber|#FBBF24|
   |본문 텍스트|--text|#E5E9F0|
   |보조 텍스트|--muted|#8A94A6|
   |구분선|--line|#2E3A57|

2. 공통 레이아웃
   * 상단 헤더 바: 좌측에 대시보드 제목 + 한 줄 설명, 우측에 데이터 기준일/출처 요약
   * 네비게이션: "한눈에 / 신규 모델 / 트렌드 / 시사점" 4개 탭(활성=Cyan)
   * 메인 영역: CSS Grid 기반 카드 배치(반응형 12컬럼 권장)
   * 카드: 카드 배경(--card), 둥근 모서리(border-radius: 12px), 은은한 그림자

3. 핵심 컴포넌트
   * 요약 KPI 카드: 핵심 수치(값=Cyan Bold) + 라벨(--muted) (예: 신규 모델 수, 평균 주행거리, 가격대 범위)
   * 모델 카드: 차량명 + 브랜드 + 주요 스펙(주행거리/배터리/가격) + 출시 시점 배지(신규=Lime, 예정=Cyan, 콘셉트=Amber)
   * 차트 카드: Chart.js 라인/바/도넛 차트, 네온톤 컬러, 다크 배경용 격자선(--line)
   * 비교 테이블: 모델별 스펙 비교, 헤더 텍스트 Cyan Bold, 행 hover 시 살짝 밝아짐
   * 시사점 박스: 카드 배경 + 좌측 Lime 세로 바 + "핵심 시사점" 라벨 + 본문

4. 대시보드 구성 (섹션 단위)
   1) 헤더 + 요약 KPI 4종 (신규 모델 수 / 평균 주행거리 / 평균 가격 / 주요 출시 지역)
   2) 신규 EV 모델 카드 그리드 (출시·예정·콘셉트별)
   3) 트렌드 차트 (연도별 주행거리 향상 추이, 가격대 분포, 브랜드별 출시 수 등)
   4) 핵심 트렌드 요약 (배터리/충전/자율주행/가격 경쟁 등 키워드 카드)
   5) 주요 시사점 (소비자·산업·경쟁 관점 3~4개 시사점 박스)

5. 인터랙션 규칙
   * 탭 클릭 시 해당 섹션으로 스크롤 또는 표시 전환(간단한 JS)
   * 모델 카드·차트 카드 hover 시 미세한 떠오름 효과(transform: translateY)
   * 출시 상태 배지는 색상으로 직관적으로 구분

6. 데이터·콘텐츠 규칙
   * 웹 검색으로 최신 EV 모델·스펙·출시 정보를 수집하고, 각 데이터에 출처/기준일을 함께 표기한다
   * 확인되지 않은 수치는 추정하지 말고 [출처/데이터 입력] 플레이스홀더로 남긴다
   * 라벨·안내 문구는 캐주얼하고 간결하게 (예: "올해 가장 핫한 EV", "지금 뜨는 흐름")
   * 강조 수치·키워드는 Cyan 또는 Lime으로 표시

7. 품질 검수
   * 다크 배경에서 텍스트 명도 대비 확인(WCAG AA 권장)
   * 반응형으로 화면 폭이 좁아져도 카드가 깨지지 않는지 확인
   * 차트가 다크 배경에서 잘 보이는지 확인
   * 수집한 데이터에 출처·기준일이 표기되었는지 확인
   * 플레이스홀더 누락 여부 점검
   * 컬러 변수·폰트 일관성 확인
